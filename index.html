# box

박스는 내용요소인 content가 있고
내용과 경계사이의 공간인 padding이 있다
padding은 처음에는 없는 공간임을 명심하자

마지막으로 박스바깥은 margin이다
margin은 블록의 레벨에따라 처음에 존재유무가 다르다
inline, inline-block은 마진이 없다
content -> 내용 요소이다
너비와 높이를 변화시키면 크기변화 하는게 content이다

블록의 레벨에 따라 크기가 다르니 알아두자

padding -> content와 border사이 공간이다
처음에는 무조건 없음을 알아두자

padding의 크기변화시 경계의 길이가 변하는 것도 중요함

padding의 크기변화시 블록의 레벨에 따라
content의 크기에도 영향을 줄 수 있으니 알아두자

border -> padding이 없으면 바로 content를 둘러싼다

border상 하 좌 우 유무를 이용하면 border중첩현상을 방지 할 수 있다

cf) outline은 외곽선인데 outline-offset속성을 음수값을 주면
외곽선을 박스 안으로 사용 할 수 있다

margin -> 박스바깥의 영역이다


양쪽 마진은 서로 중첩되지 안고 밀어낸다

상,하 마진은 서로 중첩이되어 겹치게 된다

# width,height

너비와 높이는 크기!!! 를 부여하는 매우 중요한 친구들 이다

단위는 고정값과 상대값이 있는데
무조건 그 크기만 사용하려면 고정값을 쓰면되고
화면이나 다른 요소에 대해 상대적으로 크기를 지정하고 싶으면
상대값을 사용하면 된다

상대값은 단위마다 누구를 기준으로 할지 중요하다

너비와 높이의 max,min도 매우 중요하다

max,min은 요소에 제한을 거는것으로 가장 우선시된다

우선시??!!
요소의 크기를 지정할때 여러가지 속성이 겹치게 되는 일이 발생하는데
이 경우에 누가 요소의 크기를 우선적으로 결정하냐가 포인트이다!!

# flex

그림에서 약속 하나 하고 가자!!
flex를 지정하는 파란색판을 flex판이라 하고
그안의 빨간색 상자는 box라고 하자

flex는 공간배치를 수월하게 도와주는데
flex판 안의 box크기를 지정한것 vs 크기 지정 안한것 차이가 있다
앞서 크기 지정여부가 중요하다고 강조한 이유가 이런 detail의 차이 때문이다

flex가 배치를 하기 위해서는 당연히 기준이 필요하다!!
그 기준이 바로 box의 배열 방향을 결정하는 주축
그리고 주축과 교차하는 방향의 교차축

flex박스는 크게 2가지 절차가 있다
1st. 순서
2nd. 정렬방법

주축의 방향은 가로인가 세로인가
1줄만 -> 박스의 크기합이 flex판보다 큰 경우에는 n등분이된다


2nd. 정렬방법

justify = 주축에 대한 정렬
align = 교차축에 대한 정렬

주축에 대한 정렬방법은
주로 justify-content를 사용한다

정렬의 유형은 사진 참고

교차축에 대한 정렬방법은
크게 3가지 방식을 사용한다

align-content는 justify-content와 흡사 해보인다

align-items는 3가지 방식중 많이 사용한다고 한다

왜냐, align-content와 달리 start, end, center에서
서로간 간격을 내기 좋기 때문이다

align-self는 자식에서 사용하는 속성이다

플렉스의 보라색 영역에서 각각의 줄마다 내부적으로 배치함

box의 크기를 지정하지 안은 경우는 어떨까??
물론 box안의 내용요소들 (예, 문자 )의 크기는 있다
허나, 중요한건 box공간은 별도로 크기를 주지 안은게 핵심이다

flex: 1 이라는 속성을 이용하면
flex판의 주축방향을 n등분하여 나눠줄 수 있다
근데, 이렇게 되면 1줄만 box들이 가득차게 된다

앞서 배운 min-width를 사용하면
box의 크기는 최소한도 너비를 가지게 되어서
다음줄로 밀려나게 된다

# containing block

부모는 조건이 맞아야 자식을 감쌀 수 있다
그게 아니라면 자식요소의 크기가 큰 경우
부모의 밖으로 넘치는 현상이 발생할 수 있다

containing block은 누가 부모인가를 결정 하는 것이다
가령, 크기의 상대단위인 %는 부모를 기준으로 크기를 결정하는데
무조건 상위의 요소가 부모는 아니기 때문이다
그리고 필요하다면 그 상위의 요소가 부모의 역할이 되게 만들어야지
자유자재로 배치 할 수 있다

이따 다룰거지만 position 속성을 기준으로 containing block를 정한다
position의 기본값은 static이고
이들은 우리가 흔히 알듯이 상위의 요소가 부모들이다
하지만 absolute는 static이 부모가 될 수 없어서
우리가 아는 상위요소를 부모로 보았다가는 원하는 배치를 하기 힘들어진다
그래서 공식처럼 absolute를 사용하려면 상위요소에 relative를 넣어서
그 상위요소를 부모로 만드는것이 이러한 이유이다

# block formatting context

block formatting context는 어디까지가 자식인가

이따 다룰거지만 overflow의 경우 기본값이 visible인데
visible은 자식으로 취급하지 안는다
이런경우에 자식의 요소가 부모 보다 크기가 커서 넘치는경우가 발생함
그래서 overflow를 auto혹은 scroll등으로 바꿔주는 것이다

# position

position은 요소가 위로 떠버린다
css 코드상의 위치가 아래로 갈 수록 더 위로 뜬다
z-index의 높은숫자로 조절가능함

position은 주로 자식에 적용을 하는데
절대위치와 상대위치가 있다

사진에서 중요하게 봐야할 포인트는 2가지이다!!

1. 누구를 기준으로 위치를 정하는가
2. 다른요소에 영향을 주는가

# overflow

overflow는 주로 부모에서 사용하고
자식요소가 넘치는 경우에 어떻게 처리할 것인지를 담당한다

# float

float는 요소를 띄어서 주로 왼쪽, 중간, 오른쪽에 배치하는 방법이다

float는 나머지 요소가 주위에서 감싸는 특징이 있는데
float의 속성값을 주는 순간부터
그 요소는 block formatting context가 되기 때문에
나머지 요소들도 block formatting context로 만들어주면
서로 감싸지 안게 된다

# font

글자에서 정렬 부분은 중요한 테크닉이다
가령, 네모박스안에서 글자를 정가운데에 배치하고 싶을때
text-align과 line-height를 사용 할 수 있다

# backgournd

배경은 크게 이미지와 색이 있다
배경이미지는 우리가 아는< img > 태그와는 다르다

배경이미지는 요소(예, 글자)라는 창틀 뒤에서 보여지는것이다!!

이것이 < img > 태그와 다른 점이고
공간의 크기는 요소(예, 글자)가 결정하는 것이다

배경이미지랑 요소의 크기랑 다르기떄문에
배경이미지가 모두 안보이거나
배경이미지가 반복되는 현상들이 나타는데
이를 해결하기위한 속성들이 사진의 4가지 속성들이다

# transform

마지막으로 움직임에 대한 스타일이다
요소가 그냥 움직일수도 있고
사용자와 상호작용 (클릭, 접촉)으로 움직인다

동적요소의 핵심은 시점에 따라
다른 스타일 속성을 적용하여 움직이게 보이는 것이다

transform은 요소를 변형시키는 방법을 제시한다

transition은 스타일의 속성을 시간에 따라 바꾸어서
마치 애니메이션처럼 보이게 해준다

transition을 적용할 선택자에서
어떤속성에 적용할지 정의를 해주고


적용할 선택자:행동조건에서
적용된 속성의 바뀔 값을 적으면 된다
